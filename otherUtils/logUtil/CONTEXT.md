# 开发上下文

## 当前版本

首版离线日志分析器，主文件为 `log-analyzer.html`。交付约束为单 HTML、零依赖、Chrome 125+、可通过 `file://` 直接运行。

## 已实现决策

- 视觉沿用现有 JSON 工具的紧凑 macOS 风格：默认深色、CSS 变量、`body.dark` / `body.light` 双主题、工具栏与卡片面板。
- 输入支持 `.log` / `.txt` 选择、拖放和粘贴；明确不支持 JSON Lines。
- 文件通过 2 MB `Blob.slice()` 分块读取，以流式 `TextDecoder` 处理跨块 UTF-8 字符和跨块残行。
- 解析循环每约 4000 行通过 `setTimeout(0)` 让出主线程；读取期间可以取消。
- 单文件上限为 250 MB，目标工作负载约 100 MB。超限时不读取并给出错误。
- 解析器以时间戳或级别标识新事件，无法识别为新事件的行并入上一事件，用于归并 Java 多行堆栈。
- 时间支持常见 `yyyy-MM-dd`、`yyyy/MM/dd`、无年份 `MM-dd` / `MM/dd` 格式及毫秒；无年份时使用当前年份。
- 级别支持 TRACE、DEBUG、INFO、WARN、ERROR、FATAL，并映射 WARNING 与 SEVERE。
- 异常分组键为“短异常类型 + 首个归一化业务栈帧”；归一化移除行号，并优先跳过常见框架/JDK 包。
- 日志列表固定 72px 行高，按滚动位置仅渲染可视区及上下缓冲区，避免为全部结果创建 DOM。
- localStorage 只使用 `log-analyzer-theme` 保存主题，绝不保存日志正文、筛选结果或文件内容。

## 后续可扩展项

- 增加用户自定义解析模板与时间格式。
- 支持非 UTF-8 编码选择。
- 通过 Web Worker 进一步隔离超大文件解析（仍保持内联单文件）。
- 增加异常趋势时间桶和线程/Logger 聚合。
- JSON Lines 如需支持，应作为后续明确版本能力，不应在当前启发式文本解析器中隐式处理。
