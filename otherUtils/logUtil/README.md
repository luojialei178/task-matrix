# 离线日志分析器

`log-analyzer.html` 是一个纯离线、零依赖的单文件日志分析工具，面向 Chrome 125+，可直接通过 `file://` 打开。首版支持常见文本日志与 Java 异常堆栈，不支持 JSON Lines。

## 功能

- 选择或拖放 `.log` / `.txt` 文件，也可粘贴日志文本。
- 识别常见时间、TRACE/DEBUG/INFO/WARN/ERROR/FATAL、线程、Logger/类名及消息。
- 将 Java 异常及其多行堆栈归并到同一日志事件。
- 按级别、关键字、开始时间、结束时间过滤；结果中高亮关键字。
- 统计总事件数及六种日志级别数量。
- 依据“异常类型 + 归一化首个业务栈帧”聚合异常，点击分组即可筛选。
- 结果区采用固定行高窗口化渲染，仅创建可视范围附近的 DOM。
- 使用 `Blob.slice()`、`TextDecoder` 分块读取文件，展示进度并定期让出主线程。
- 支持取消读取、超大文件拦截、导出过滤结果和一键清空。
- 默认深色主题，可切换浅色主题；适配窄屏。

## 使用

直接双击 `log-analyzer.html`，或在 macOS 中执行：

```bash
open -a "Google Chrome" log-analyzer.html
```

选择文件或粘贴文本后开始分析。点击日志行可在右侧查看完整正文；点击异常分组可只看该类异常。

## 解析说明

日志格式很多，工具采用宽松的启发式规则：以可识别的时间或日志级别判断新事件，后续未识别为新事件的行归入上一事件，因此适合 Logback、Log4j、JUL 等常见纯文本输出。缺少时间的事件在启用时间范围过滤时会被排除。`WARNING` 映射为 `WARN`，`SEVERE` 映射为 `ERROR`。

异常业务栈帧会优先跳过 JDK、Spring、Apache、Reactor、Kotlin 等常见框架包；若没有业务帧，则使用首个栈帧。归一化时移除源码行号，降低同一异常因行号变化而拆组的概率。

## 性能与限制

- 目标场景约 100 MB；为避免浏览器内存失控，单文件硬限制为 250 MB。
- 文件读取按 2 MB 分块，但解析后的事件仍需保存在当前页面内存中；实际容量受机器内存、单行长度和异常堆栈数量影响。
- 导出大量过滤结果时，浏览器会临时占用额外内存。
- 首版不解析 JSON Lines，也不进行 JSON 字段提取。
- 文本按 UTF-8 解码；其他编码应先转换为 UTF-8。

## 隐私

所有读取、解析、筛选和导出均在浏览器本地完成，不发起网络请求。日志正文不会写入 `localStorage`；其中仅保存主题偏好键 `log-analyzer-theme`。关闭或刷新页面后，已载入日志不会恢复。
