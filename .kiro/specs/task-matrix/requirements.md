# 任务优先级矩阵工具 — 需求文档

## 项目概述

本工程专门维护一个单文件桌面小工具 `task-matrix.html`，零依赖，浏览器直接打开使用。  
工具用于对任务按"任务优先级"（纵轴）和"事项优先级"（横轴）进行二维矩阵分类管理。

---

## 术语

| 术语 | 说明 |
|------|------|
| 行 | 矩阵纵轴，代表任务优先级（P1 最高 → P5 最低） |
| 列 | 矩阵横轴，代表事项优先级（P1 最高 → P5 最低，从左到右） |
| 单元格 | 行列交叉格子，存放任务卡片 |
| 任务卡片 | 单元格内的一条任务记录 |
| 扩展行/列 | 用户动态添加的额外行或列，作最低优先级记录用 |
| 逻辑行 | 任务数据存储层的行索引（localStorage key 中的 r0~rN） |
| 视觉行 | 页面渲染层的行位置（颜色按视觉位置固定） |

---

## 核心需求

### 需求 1：矩阵布局

**User Story:** 作为用户，我希望以二维矩阵查看任务，使我可以直观判断任务的综合优先级。

#### 验收标准

1. 默认显示 5×5 矩阵，纵轴从上到下为 P1→P5（任务优先级高到低），横轴从左到右为 P1→P5（事项优先级高到低）
2. 纵轴左侧显示文字标签"任务优先级 高 → 低"（竖排，从上到下可读）
3. 横轴上方显示文字标签"事项优先级：高 → 低"
4. 行标题颜色按视觉位置固定：P1=红(#ff3b30)、P2=橙(#ff9500)、P3=黄(#ffcc00)、P4=绿(#34c759)、P5=蓝(#007aff)，扩展行=灰
5. 列标题颜色与行标题一一对应
6. 单元格背景为热力图色调：行列均为高优先级的格子颜色最深，均为低的最浅

---

### 需求 2：任务管理

**User Story:** 作为用户，我希望在单元格内添加、完成和删除任务，使我可以实时维护任务清单。

#### 验收标准

1. 点击单元格内"＋ 添加任务"按钮，出现输入框；回车确认添加，Esc 取消
2. 输入框失焦超过 150ms 自动取消
3. 任务卡片左侧有复选框，点击切换完成状态：完成时显示绿色 ✓，文字加删除线，卡片背景变为完成色
4. 任务卡片右侧有 × 删除按钮，点击立即删除
5. 有任务的单元格显示实线边框，空格子显示虚线边框
6. 任务文字支持自动换行，单元格宽度随列宽动态变化
7. 任务卡片背景继承所在单元格的颜色色调（深色模式：更深的同色调；浅色模式：淡色同色调）

---

### 需求 3：拖拽功能

**User Story:** 作为用户，我希望通过拖拽移动任务和调整行顺序，使我可以灵活重新组织任务。

#### 验收标准

1. 任务卡片可拖拽到任意其他单元格，松手后任务移入目标单元格
2. 行标题（P1~P5 色块）可拖拽，拖到另一行标题上松手后两行数据和名称互换
3. 行颜色按视觉位置固定，不随拖拽改变
4. 行拖拽和任务卡片拖拽互不冲突
5. 行标题拖拽时显示半透明状态（opacity 0.35），目标行显示蓝色高亮轮廓

---

### 需求 4：行列扩展

**User Story:** 作为用户，我希望动态添加和删除扩展行列，使我可以记录超出 5 个优先级的任务。

#### 验收标准

1. 矩阵右侧有 ＋ 按钮，点击添加扩展列，标签为"扩展1"、"扩展2"...，颜色为灰色
2. 矩阵下方有 ＋ 按钮，点击添加扩展行，同上
3. 扩展列/行的标题右上角有 × 删除按钮，点击弹出确认后删除该列/行及其所有任务
4. 删除扩展列/行后，后续列/行自动向前补位，数据不丢失
5. 基础 P1~P5 行列不显示删除按钮，不可删除

---

### 需求 5：行标题编辑

**User Story:** 作为用户，我希望自定义行标题名称，使我可以用具体业务语义替代 P1~P5。

#### 验收标准

1. 双击任意行标题（包括扩展行），出现内联输入框
2. 回车确认，Esc 取消；失焦自动提交
3. 名称最长 12 个字符
4. 自定义名称持久化到 localStorage，刷新后保留
5. 列标题不支持编辑（固定为 P1~P5 和扩展标签）

---

### 需求 6：主题切换

**User Story:** 作为用户，我希望在深色和浅色模式间切换，使我可以适应不同光线环境。

#### 验收标准

1. Header 标题右侧有一个小圆点按钮：深色模式下白色，浅色模式下黑色
2. 点击圆点切换主题
3. 深色主题背景色 `#1c1c1e`（macOS 系统深色），浅色主题背景色 `#f2f2f7`（iOS 系统浅灰）
4. Header 和 Footer 使用毛玻璃效果（backdrop-filter blur）
5. 主题偏好存储到 localStorage，刷新后保持上次选择
6. 默认主题为深色

---

### 需求 7：数据持久化与导入导出

**User Story:** 作为用户，我希望数据自动保存并支持导入导出，使我可以在版本迭代时同步数据。

#### 验收标准

1. 所有数据（任务、行名称、行顺序）自动存储到 localStorage，刷新不丢失
2. 点击"导出 JSON"将当前完整数据（state / rowNames / rowOrder）下载为 `task-matrix.json`
3. 点击"导入 JSON"弹出文件选择，选择之前导出的 json 文件后弹出确认提示，确认后覆盖导入
4. 导入失败时显示错误提示，不损坏现有数据
5. 导入后重新渲染矩阵，数据完整还原
6. Header 按钮顺序：清除已完成 · 导出 JSON · 导入 JSON

---

### 需求 8：清除已完成

**User Story:** 作为用户，我希望批量清除已完成的任务，使我可以保持矩阵整洁。

#### 验收标准

1. 点击"清除已完成"按钮，弹出确认对话框
2. 确认后删除所有 `done: true` 的任务卡片，重新渲染矩阵
3. 取消则不做任何操作

---

### 需求 9：视觉与交互规范

**User Story:** 作为用户，我希望工具风格与 macOS 一致，使我有原生应用的使用体验。

#### 验收标准

1. 使用系统字体栈：`-apple-system, BlinkMacSystemFont, "SF Pro Text", "Segoe UI"`
2. 圆角层级：按钮 6px，单元格 10px，行/列标题 8px
3. Header 高度 44px，与 macOS 窗口标题栏一致
4. 按钮三级视觉层次：清除已完成（最浅）< 导出/导入（稍深）< 强调按钮
5. 深色模式使用 iOS/macOS 系统色：红 #ff3b30、橙 #ff9500、黄 #ffcc00、绿 #34c759、蓝 #007aff
6. 滚动条宽度 4px，半透明，不影响布局
7. 工具为单文件 HTML，零外部依赖，Chrome 125+ 可直接打开使用
